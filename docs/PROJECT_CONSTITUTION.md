# 2. Принципы и правила перевода

## 2.1 Фундаментальные принципы

### Принцип 1: Верность оригиналу (الأمانة)
- Перевод должен точно передавать **смысл** арабского текста
- Запрещено добавление собственных толкований без пометки `[Прим. пер.]`
- Неясные места сохраняются как неясные, с примечанием

### Принцип 2: Естественность языка
- Текст должен читаться естественно на целевом языке
- Избегать кальки с арабского синтаксиса
- Длинные арабские предложения разбиваются на несколько

### Принцип 3: Единообразие терминологии
- Один арабский термин = одно устоявшееся соответствие на целевом языке
- Все отклонения фиксируются в глоссарии с обоснованием

### Принцип 4: Доступность
- Текст рассчитан на широкую аудиторию, не только на учёных
- Сложные понятия сопровождаются краткими пояснениями
- Текст должен быть пригоден для восприятия на слух (аудиокнига)

### Принцип 5: Адаб (أدب) — Этикет
- Обязательные формулы почтения: ﷺ, عليه السلام, رضي الله عنه
- В переводе: «мир ему и благословение Аллаха», «мир ему», «да будет доволен им Аллах»
- Коранические аяты выделяются и приводятся с номером суры и аята

## 2.4 Оформление коранических аятов

```
Формат: «Текст аята на целевом языке» (Название суры, номер аята)

Пример (русский):
«И Мы отправили тебя только как милость для миров» (Аль-Анбия, 107)

Пример (узбекский):
«Биз сени барча оламларга раҳмат қилиб юбордик» (Анбиё, 107)

Пример (таджикский):
«Ва Мо туро ҷуз раҳмат барои ҷаҳониён нафиристодаем» (Анбиё, 107)
```

## 2.5 Оформление хадисов

```
Формат:
Передатчик (رضي الله عنه/رضي الله عنها) сообщил, что Посланник Аллаха ﷺ сказал:
«Текст хадиса»
(Источник: Аль-Бухари, №XXXX / Муслим, №XXXX)

Степень достоверности указывается, если автор (Ибн Касир) её упоминает.
```

## 2.6 Обработка иснадов

Ибн Касир часто приводит длинные цепочки передатчиков. Стратегия:

| Режим | Описание | Когда использовать |
|-------|----------|-------------------|
| **Полный** | Весь иснад переводится полностью | Ключевые хадисы, где иснад важен для аргументации |
| **Краткий** | «Передаётся от [первый передатчик] через [последний перед автором]» | Большинство случаев |
| **Минимальный** | «Передаётся от [сподвижник]» | Повторяющиеся хадисы, известные версии |

> **Решение о режиме принимает агент-редактор** с учётом контекста.

## 2.7 Обработка исраилийят

Ибн Касир включает некоторые истории из иудейско-христианских источников. Правила:
1. Если автор критикует повествование — перевести критику полностью
2. Если автор приводит без комментариев — добавить `[Прим. пер.: автор приводит это повествование как исраилийят без подтверждения достоверности]`
3. Если автор подтверждает согласие с Кораном — перевести как есть

## 3.2 Роли агентов

### 🔵 КООРДИНАТОР (Orchestrator)

**Назначение:** Центральный узел управления всем процессом.

**Обязанности:**
- Разбивка тома на рабочие единицы (главы → разделы)
- Назначение задач агентам с приоритетами
- Мониторинг прогресса и дедлайнов
- Разрешение конфликтов между агентами
- Формирование отчётов о статусе проекта
- Управление версиями и ветками

**Входные данные:** Арабский текст (размеченный по главам), конфигурация проекта  
**Выходные данные:** Задачи для агентов, отчёты, метрики

### 🟢 ПЕРЕВОДЧИК (Translator)

**Экземпляры:** 3 (по одному на язык: RU, UZ, TJ)

**Обязанности:**
- Первичный перевод с арабского на целевой язык
- Следование глоссарию и терминологической базе
- Пометка неясных мест тегом `{{UNCLEAR: описание}}`
- Пометка мест, требующих проверки: `{{VERIFY: описание}}`
- Сверка с английским переводом Darussalam при сложных пассажах

**Входные данные:** Арабский текст раздела, глоссарий, контекст (предыдущие разделы)  
**Выходные данные:** Черновой перевод в формате Markdown

**Ограничения:**
- НЕ принимает решений по шариатским вопросам
- НЕ добавляет собственных комментариев (только `[Прим. пер.]` с пометкой)
- НЕ пропускает текст оригинала

### 🟡 РЕДАКТОР (Editor)

**Экземпляры:** 3 (по одному на язык)

**Обязанности:**
- Стилистическая правка перевода
- Проверка естественности языка
- Унификация стиля в пределах тома
- Проверка полноты (ничего не пропущено)
- Разрешение тегов `{{UNCLEAR}}` — либо уточнение, либо эскалация верификатору
- Адаптация длины предложений для аудиовосприятия

**Входные данные:** Черновой перевод, оригинал, глоссарий  
**Выходные данные:** Отредактированный перевод, список замечаний

### 🟠 ТЕРМИНОЛОГ (Terminologist)

**Экземпляры:** 1 (работает со всеми тремя языками)

**Обязанности:**
- Ведение и обновление глоссария
- Проверка единообразия терминологии во всех переводах
- Исследование устоявшихся переводов терминов
- Согласование новых терминов с верификатором
- Поддержка таблиц соответствия между тремя языками

**Входные данные:** Новые термины из переводов, существующий глоссарий  
**Выходные данные:** Обновлённый глоссарий, рекомендации по терминологии

### 🔴 ВЕРИФИКАТОР (Shariah Reviewer)

**Экземпляры:** 1

**Обязанности:**
- Проверка соответствия перевода вероучению Ахль ас-Сунна валь-Джама'а
- Верификация переводов коранических аятов
- Проверка точности перевода хадисов
- Проверка правильности атрибуции (кто сказал, источник)
- Валидация обработки исраилийят
- Разрешение тегов `{{VERIFY}}`

**Входные данные:** Отредактированный перевод, арабский оригинал, тафсир, сборники хадисов  
**Выходные данные:** Верифицированный перевод, список правок, теологические примечания

### 🟣 ФОРМАТИРОВЩИК (Formatter)

**Экземпляры:** 1

**Обязанности:**
- Приведение текста к финальному формату
- Генерация аудио-разметки (см. раздел 8)
- Создание оглавления и навигационной структуры
- Вставка метаданных
- Подготовка для публикации (PDF, EPUB, аудио)

**Входные данные:** Верифицированный перевод  
**Выходные данные:** Финальный документ со всей разметкой

### ⚫ QA-АГЕНТ (Quality Gate)

**Экземпляры:** 1

**Обязанности:**
- Финальная проверка по чек-листу (раздел 6)
- Прогон автоматических проверок (форматирование, ссылки, нумерация)
- Сравнение объёмов (оригинал vs перевод — допустимое отклонение ±20%)
- Проверка отсутствия незакрытых тегов `{{UNCLEAR}}`, `{{VERIFY}}`
- Валидация аудио-разметки

**Входные данные:** Финальный документ  
**Выходные данные:** QA-отчёт (pass/fail), список критических замечаний

## 3.3 Workflow — Основной процесс

```
ЭТАП 1: ПОДГОТОВКА
  Координатор → Разбивка главы на разделы (5–10 стр. оригинала)
  Координатор → Извлечение новых терминов → Терминолог
  Терминолог  → Обновление глоссария

ЭТАП 2: ПЕРЕВОД
  Координатор → Раздел + Глоссарий → Переводчик (RU/UZ/TJ)
  Переводчик  → Черновой перевод → Координатор

ЭТАП 3: РЕДАКТУРА
  Координатор → Черновик → Редактор (соответствующий язык)
  Редактор    → Отредактированный текст → Координатор

ЭТАП 4: ВЕРИФИКАЦИЯ
  Координатор → Отредактированный текст → Верификатор
  Верификатор → Верифицированный текст → Координатор

ЭТАП 5: ФОРМАТИРОВАНИЕ
  Координатор → Верифицированный текст → Форматировщик
  Форматировщик → Финальный документ → Координатор

ЭТАП 6: QA
  Координатор → Финальный документ → QA-Агент
  QA-Агент    → PASS → Публикация
  QA-Агент    → FAIL → Возврат на соответствующий этап
```

## 3.4 Протоколы коммуникации

### Формат передачи задачи

```yaml
task:
  id: "vol03_ch05_sec02_ru"
  type: "translation"          # translation | editing | verification | formatting | qa
  source_file: "arabic/vol03/ch05/sec02.md"
  target_language: "ru"
  assigned_to: "translator_ru"
  assigned_by: "orchestrator"
  priority: "normal"           # critical | high | normal | low
  deadline: "2026-06-15"
  dependencies:
    - "glossary_update_vol03_ch05"
  context:
    previous_section: "vol03_ch05_sec01_ru"
    glossary_version: "1.42"
  notes: "Содержит длинный иснад — применить краткий режим"
```

## 4.3 Version Control

### Git Workflow

1. **Каждый раздел** — отдельный коммит
2. **Формат коммита:**
   ```
   [{stage}][{lang}] vol{NN} ch{NN} sec{NN}: краткое описание
   
   Примеры:
   [draft][ru] vol01 ch01 sec01: первичный перевод
   [edit][ru] vol01 ch01 sec01: стилистическая правка, 12 замечаний
   [verify][ru] vol01 ch01 sec01: проверка хадисов, 2 исправления
   [final][ru] vol01 ch01 sec01: финальная версия
   [audio][ru] vol01 ch01 sec01: аудио-разметка добавлена
   ```

3. **Pull Request** обязателен для перехода `draft → edited → verified → final`
4. **Теги версий:** `v{том}.{глава}.{ревизия}` — `v01.01.0`, `v01.01.1`

# 6. Quality Assurance

## 6.1 Чек-лист проверки перевода

### A. Полнота (Completeness)

- [ ] Весь текст оригинала переведён (ничего не пропущено)
- [ ] Все коранические аяты присутствуют с номерами
- [ ] Все хадисы присутствуют с указанием источников
- [ ] Все имена собственные переведены/транслитерированы
- [ ] Все даты (хиджра и григорианские) присутствуют
- [ ] Все примечания автора (Ибн Касира) сохранены
- [ ] Метаданные YAML-фронтматтера заполнены полностью

### B. Точность (Accuracy)

- [ ] Перевод соответствует смыслу арабского оригинала
- [ ] Номера аятов и сур верны
- [ ] Номера хадисов верны
- [ ] Имена передатчиков корректны
- [ ] Исторические даты корректны
- [ ] Степени достоверности хадисов указаны верно
- [ ] Генеалогии и нисбы (отчества) корректны

### C. Терминология (Terminology)

- [ ] Все термины соответствуют глоссарию
- [ ] Нет «плавающей» терминологии (разные переводы одного термина)
- [ ] Формулы почтения присутствуют при каждом упоминании
- [ ] Арабские термины транслитерированы единообразно
- [ ] При первом упоминании дано арабское написание в скобках

### D. Язык и стиль (Language & Style)

- [ ] Текст читается естественно на целевом языке
- [ ] Нет калек с арабского синтаксиса
- [ ] Нет орфографических ошибок
- [ ] Нет грамматических ошибок
- [ ] Пунктуация корректна
- [ ] Стиль единообразен в пределах раздела/главы

### E. Форматирование (Formatting)

- [ ] Коранические аяты оформлены по стандарту
- [ ] Хадисы оформлены по стандарту
- [ ] Примечания переводчика маркированы `[Прим. пер.]`
- [ ] Заголовки имеют правильную иерархию
- [ ] HTML-комментарии разметки корректны и закрыты
- [ ] Нет незакрытых тегов `{{UNCLEAR}}` или `{{VERIFY}}`

### F. Аудио-готовность (Audio-readiness)

- [ ] Предложения не длиннее 25 слов (для комфортного восприятия)
- [ ] Нет аббревиатур без расшифровки
- [ ] Числа записаны словами (для озвучки)
- [ ] Сложные арабские имена имеют фонетическую подсказку
- [ ] Паузы размечены (`<!-- PAUSE -->`)

## 6.2 Критерии оценки качества

Шкала оценки: **0–100 баллов**

| Критерий | Вес | Описание |
|----------|-----|----------|
| Полнота | 25% | Весь оригинал переведён без пропусков |
| Смысловая точность | 30% | Точная передача смысла оригинала |
| Терминологическое единообразие | 15% | Соответствие глоссарию |
| Языковая естественность | 15% | Качество языка перевода |
| Форматирование | 10% | Соблюдение стандартов разметки |
| Аудио-готовность | 5% | Пригодность для озвучки |

**Пороги:**
- **90–100:** Отлично — готов к публикации
- **80–89:** Хорошо — минимальная доработка
- **70–79:** Удовлетворительно — требуется доработка
- **< 70:** Неудовлетворительно — возврат на переработку

## 6.3 Процесс Review

```
1. SELF-REVIEW (Переводчик)
   - Переводчик перечитывает свой текст перед отправкой
   - Прогоняет автоматическую проверку глоссария
   
2. PEER-REVIEW (Редактор)
   - Полная вычитка на целевом языке
   - Сверка случайных 10% с оригиналом
   - Оценка по чек-листу (разделы D, E)
   
3. EXPERT-REVIEW (Верификатор)
   - Проверка шариатской точности
   - Сверка аятов и хадисов с первоисточниками
   - Оценка по чек-листу (разделы A, B, C)
   
4. FINAL QA (QA-Агент)
   - Автоматические проверки
   - Полный чек-лист
   - Выставление финальной оценки
   - Решение: PASS / FAIL с указанием причин
```

### Политика возвратов

| Оценка QA | Действие |
|-----------|----------|
| PASS (≥80) | Публикация |
| FAIL из-за терминологии | Возврат Терминологу → Переводчику |
| FAIL из-за смысла | Возврат Верификатору → Переводчику |
| FAIL из-за языка | Возврат Редактору |
| FAIL из-за формата | Возврат Форматировщику |

# 7. Этапы проекта

## 7.1 Этап 0: Инициализация (2–3 недели)

| Задача | Ответственный | Результат |
|--------|--------------|-----------|
| Утверждение PROJECT_CONSTITUTION | Координатор | Финальный документ |
| Настройка репозитория | Координатор | Git-репозиторий |

| Настройка автоматических проверок | QA-Агент | Скрипты в `tools/` |
| Тестирование workflow на мини-отрывке | Все агенты | Отчёт о готовности |

## 7.2 Этап 1: Pilot — Тестовая глава (3–4 недели)

**Цель:** Проверить весь pipeline на одной главе, выявить проблемы, откалибровать процесс.

**Выбор главы:** Рекомендуется взять главу средней сложности — не самую лёгкую и не самую сложную. Например, раздел о жизнеописании одного из пророков (аль-Анбия).

| Неделя | Задача | Языки |
|--------|--------|-------|
| 1 | Перевод тестовой главы | RU, UZ, TJ параллельно |
| 2 | Редактура + первичная верификация | RU, UZ, TJ |
| 3 | Полный QA + исправление замечаний | Все |
| 4 | Ретроспектива и обновление конституции | — |

**Результаты пилота:**
- Откалиброванный глоссарий
- Уточнённые оценки скорости перевода
- Обновлённые правила (если нужно)
- Первая опубликованная глава

## 7.3 Этап 2: Production — Основной перевод (12–18 месяцев)

### Стратегия перевода

**Порядок томов:** Последовательный (том 1 → том 14), но с возможностью параллельной работы над 2–3 томами при наличии ресурсов.

**Примерный темп:**
- 1 раздел (~5–10 стр. оригинала) = 1–2 рабочих дня на язык
- 1 глава = 1–2 недели
- 1 том = 4–8 недель
- Весь проект (14 томов × 3 языка) = 12–18 месяцев

### Спринты

Работа организуется **двухнедельными спринтами:**

```
Спринт N:
  - Цель: Том X, главы Y–Z на трёх языках
  - Начало: Дата
  - Конец: Дата
  - Ежедневный отчёт: прогресс по разделам
  - Ретроспектива: последний день спринта
```

### Контрольные точки (Milestones)

| Milestone | Критерий | Действие |
|-----------|----------|----------|
| M1: Пилот завершён | Тестовая глава опубликована на 3 языках | Переход к Production |
| M2: Первый том | Том 1 полностью переведён, проверен, опубликован | Фиксация темпа |
| M3: 25% проекта | 3–4 тома завершены | Промежуточный аудит |
| M4: 50% проекта | 7 томов завершены | Полная ревизия процесса |
| M5: 75% проекта | 10–11 томов завершены | Начало аудио-производства для готовых томов |
| M6: 100% перевод | Все 14 томов переведены | Начало финального QA |
| M7: Релиз | Все тома опубликованы + аудиокниги | Завершение проекта |

## 7.4 Этап 3: Review & Audio (параллельно с Этапом 2)

Начинается после завершения первых 3–4 томов:

1. **Сквозная ревизия** — проверка единообразия терминологии между томами
2. **Аудио-подготовка** — генерация аудио-разметки (SSML)
3. **TTS-генерация** — создание аудиофайлов
4. **Аудио-QA** — прослушивание и правка произношений
5. **Публикация** — релиз аудиокниг

# 8. Аудиокниги — требования и разметка

## 8.1 Требования к тексту для озвучки

### Общие принципы

1. **Длина предложений:** Не более 25 слов. Длинные предложения разбиваются.
2. **Числа:** Записываются словами: `двести двенадцатый год хиджры`, не `212 г.х.`
3. **Аббревиатуры:** Всегда расшифровываются: `мир ему и благословение Аллаха`, не `ﷺ`
4. **Скобки и ссылки:** Убираются или озвучиваются естественно
5. **Примечания переводчика:** Предваряются фразой «Примечание переводчика:»
6. **Сноски:** Включаются в основной текст или читаются отдельным блоком в конце главы

### Структура для навигации

Каждый аудиофайл соответствует одному **разделу** (section). Навигационная структура:

```
Аудиокнига: "Аль-Бидая ва-н-Нихая"
└── Том 1: "Начало творения"
    ├── Глава 1: "О сотворении Трона и Курси"
    │   ├── Раздел 1 (трек 01-01-01) — 12:34
    │   ├── Раздел 2 (трек 01-01-02) — 08:45
    │   └── ...
    ├── Глава 2: "О сотворении небес и земли"
    │   ├── Раздел 1 (трек 01-02-01) — 15:20
    │   └── ...
    └── ...
```

**ID аудиотрека:** `{vol}-{ch}-{sec}` → `01-01-01`, `01-01-02`, ...

## 8.3 SSML-шаблон для TTS

```xml
  </p>
  
  <break time="1000ms"/>
  
  <p>
    <s>Всевышний Аллах сказал:</s>
  </p>
  
  <break time="500ms"/>
  
  <prosody rate="slow" pitch="+5%">
    <p>
      <s>Аллах — нет божества, достойного поклонения, кроме Него, Живого, Вседержителя.</s>
      <s>Им не овладевают ни дремота, ни сон.</s>
      <s>Ему принадлежит то, что на небесах, и то, что на земле.</s>

  <break time="500ms"/>
  
  <p>
    <s>Сура Аль-Бакара, аят двести пятьдесят пять.</s>
  </p>
  
  <break time="1000ms"/>
```
