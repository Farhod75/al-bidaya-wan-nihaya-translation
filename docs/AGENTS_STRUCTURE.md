## 1. Обзор архитектуры

Система состоит из **13 специализированных агентов**, организованных в три уровня:

```
┌─────────────────────────────────────────────────────┐
│              КООРДИНАТОР ПРОЕКТА                     │
│         (управление, распределение, контроль)        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │
│  │  РУССКИЙ     │ │  УЗБЕКСКИЙ  │ │ ТАДЖИКСКИЙ  │   │
│  │  ├─Переводчик│ │  ├─Переводчик│ │ ├─Переводчик│  │
│  │  ├─Редактор  │ │  ├─Редактор  │ │ ├─Редактор  │  │
│  │  └─Терминолог│ │  └─Терминолог│ │ └─Терминолог│  │
│  └─────────────┘ └─────────────┘ └─────────────┘   │
│                                                      │
├─────────────────────────────────────────────────────┤
│  ВЕРИФИКАТОР    │    QA АГЕНТ    │  АУДИО-АГЕНТ     │
│  исламского     │   (финальная   │  (подготовка     │
│  контента       │    проверка)   │   к озвучке)     │
└─────────────────────────────────────────────────────┘
```

## 2. Языковые агенты

### 2.1 Русский язык

#### 2.1.1 Агент-переводчик (RU-TRANSLATOR)

**Роль:** Первичный перевод арабского текста на русский язык.

**Задачи:**
- Перевод сегментов текста с арабского на русский с максимальным сохранением смысла
- Транслитерация арабских терминов по устоявшейся русскоязычной системе
- Маркировка неоднозначных фрагментов для ревью редактором
- Сохранение иснадов (цепочек передатчиков) в стандартизированном формате
- Корректная передача аятов Корана с указанием суры и аята
- Передача хадисов с указанием источника (сборник, номер)

**Требования:**
- System prompt должен содержать: контекст книги, стилистические указания, глоссарий базовых терминов
- Формат вывода: Markdown с метаданными (номер тома, главы, страницы)
- Температура: 0.3 (минимальная креативность, максимальная точность)
- Обязательная обработка: формулы благопожеланий (صلى الله عليه وسلم → ﷺ или «мир ему и благословение Аллаха»)

**Входные данные:**
```
- Арабский текст сегмента (500–1000 слов)
- Контекст: номер тома, главы, предыдущий переведённый сегмент
- Глоссарий терминов (из агента-терминолога)
```

**Выходные данные:**
```
- Переведённый текст в Markdown
- Список использованных терминов с арабскими оригиналами
- Флаги неопределённости: [REVIEW_NEEDED], [TERM_CHECK], [HADITH_VERIFY]
```

---

#### 2.1.2 Агент-редактор (RU-EDITOR)

**Роль:** Стилистическая и смысловая редактура русского перевода.

**Задачи:**
- Проверка литературного качества русского текста
- Обеспечение единообразия стиля на протяжении всего произведения
- Устранение кальковых конструкций с арабского
- Проверка согласованности терминологии с глоссарием
- Адаптация длинных арабских предложений к нормам русского синтаксиса
- Сохранение научного, но доступного стиля изложения

**Требования:**
- System prompt: стилистический гайд, примеры эталонных переводов исламских текстов на русский
- Доступ к предыдущим отредактированным главам для поддержания единого стиля
- Температура: 0.4

**Входные данные:**
```
- Черновой перевод от RU-TRANSLATOR
- Арабский оригинал для сверки
- Стилистический гайд проекта
- Терминологический глоссарий
```

**Выходные данные:**
```
- Отредактированный текст
- Лог изменений (diff) с обоснованием каждой правки
- Рекомендации для терминолога (при обнаружении несогласованности)
```

---

#### 2.1.3 Агент-терминолог (RU-TERMINOLOGIST)

**Специализация:** Исламская терминология на русском языке.

**Задачи:**
- Ведение и обновление русско-арабского глоссария исламских терминов
- Стандартизация транслитерации арабских имён и терминов
- Выбор между транслитерацией и смысловым переводом для каждого термина
- Обеспечение единообразия именования сахабов, учёных, географических объектов
- Согласование с устоявшейся русскоязычной исламской традицией

**Ключевые категории терминов:**
| Категория | Примеры | Подход |
|-----------|---------|--------|
| Столпы ислама | салят, закят, хадж | Транслитерация + пояснение при первом упоминании |
| Имена Аллаха | ар-Рахман, аль-Малик | Транслитерация + перевод в скобках |
| Исторические лица | Абу Бакр, Умар ибн аль-Хаттаб | Устоявшаяся русская форма |
| Географические названия | Мадина, Макка | Русская транслитерация (не англ.) |
| Правовые термины | фикх, фетва, иджма | Транслитерация |
| Богословские понятия | таухид, ширк, бида | Транслитерация + при первом упоминании перевод |

**Выходные данные:**
```
- Обновлённый глоссарий (JSON/CSV)
- Карточки терминов: арабский → транслитерация → перевод → контекст употребления
- Отчёт о терминологических решениях с обоснованием
```

### 2.2 Узбекский язык

#### 2.2.1 Агент-переводчик (UZ-TRANSLATOR)

**Роль:** Перевод арабского текста на узбекский язык (латиница).

**Задачи:**
- Перевод с арабского на узбекский с учётом специфики тюркской грамматики
- Использование современной узбекской латиницы (с 2021 года — обновлённый алфавит)
- Учёт особенностей узбекской исламской литературной традиции
- Корректная передача арабских имён в узбекской транслитерации
- Маркировка фрагментов, требующих дополнительной верификации

**Требования:**
- System prompt: контекст узбекской исламской литературной традиции, особенности узбекского богословского языка
- Учитывать наследие Мавераннахра и местные исламские традиции
- Температура: 0.3
- Алфавит: латиница (новая узбекская орфография)

**Специфика узбекского перевода:**
- Порядок слов SOV (подлежащее — дополнение — сказуемое)
- Аффиксальная система: правильное использование падежных окончаний
- Уважительные формы обращения к пророкам и сахабам
- Традиции перевода Корана на узбекский (ориентир: перевод Алоуддина Мансура)

**Входные данные:**
```
- Арабский текст сегмента
- Контекст главы
- Терминологический глоссарий (UZ)
- (опционально) Русский перевод для кросс-проверки
```

---

#### 2.2.2 Агент-редактор (UZ-EDITOR)

**Роль:** Стилистическая и грамматическая редактура узбекского перевода.

**Задачи:**
- Проверка грамматической корректности узбекского текста
- Обеспечение естественности звучания на узбекском языке
- Устранение арабизмов, не прижившихся в узбекском
- Адаптация к современному литературному узбекскому языку
- Проверка правильности латинской графики

**Требования:**
- Знание норм современного узбекского литературного языка
- Доступ к корпусу узбекских исламских текстов для стилистической проверки
- Температура: 0.4

---

#### 2.2.3 Агент-терминолог (UZ-TERMINOLOGIST)

**Специализация:** Исламская терминология на узбекском языке.

**Задачи:**
- Ведение узбекско-арабского глоссария
- Согласование с традицией узбекских исламских переводов
- Учёт терминологии, используемой Духовным управлением мусульман Узбекистана
- Стандартизация имён собственных в узбекской латинице

**Ключевые особенности:**
| Аспект | Решение |
|--------|---------|
| Арабские термины | Адаптация к узбекской фонетике: «салот» вместо «салят» |
| Имена | Узбекская форма: «Муҳаммад» → «Muhammad» (латиница) |
| Географические названия | Местная традиция: «Makka», «Madina» |
| Религиозные формулы | Стандарт: «sollallohu alayhi vasallam» |

### 2.3 Таджикский язык

#### 2.3.1 Агент-переводчик (TJ-TRANSLATOR)

**Роль:** Перевод арабского текста на таджикский язык (кириллица).

**Задачи:**
- Перевод с арабского на таджикский с учётом персидского литературного наследия
- Использование таджикской кириллицы
- Учёт богатой персо-арабской лексики в таджикском языке
- Использование преимущества близости таджикского к персидскому при переводе арабских текстов
- Передача поэтических фрагментов с сохранением ритмики

**Требования:**
- System prompt: таджикская литературная традиция, связь с персидским наследием
- Учёт ханафитской правовой школы, доминирующей в регионе
- Температура: 0.3

**Специфика таджикского перевода:**
- Богатый арабо-персидский словарный пласт — многие термины не требуют транслитерации
- Изафетная конструкция: «китоби муқаддас» (священная книга)
- Кириллическая графика с дополнительными буквами (ғ, ӣ, қ, ӯ, ҳ, ҷ)
- Традиция переводов из персидских источников (Тафсир Табари на фарси и др.)

---

#### 2.3.2 Агент-редактор (TJ-EDITOR)

**Роль:** Стилистическая и грамматическая редактура таджикского перевода.

**Задачи:**
- Проверка литературного качества таджикского текста
- Баланс между книжным и разговорным стилем
- Проверка использования таджикской кириллицы
- Устранение русизмов и узбекизмов в тексте
- Обеспечение доступности текста для современного таджикского читателя

**Требования:**
- Знание литературного таджикского языка и его богословского регистра
- Доступ к таджикским исламским изданиям для стилистической справки
- Температура: 0.4

---

#### 2.3.3 Агент-терминолог (TJ-TERMINOLOGIST)

**Специализация:** Исламская терминология на таджикском языке.

**Задачи:**
- Ведение таджикско-арабского глоссария
- Использование естественных персо-арабских заимствований в таджикском
- Согласование с терминологией Совета улемов Таджикистана
- Стандартизация имён собственных в таджикской кириллице

**Ключевые особенности:**
| Аспект | Решение |
|--------|---------|
| Арабские термины | Через персидскую адаптацию: «намоз» (а не «салят»), «рӯза» (а не «саум») |
| Имена | Таджикская форма: «Муҳаммад», «Абӯбакр» |
| Коранические термины | Персо-арабская форма: «Қуръон», «сура», «оят» |
| Религиозные формулы | «саллаллоҳу алайҳи ва саллам» |

## 3. Общие агенты

### 3.1 Координатор проекта (PROJECT-COORDINATOR)

**Роль:** Центральный агент управления всем процессом перевода.

**Задачи:**
- Разбиение текста на сегменты оптимального размера (500–1000 слов)
- Распределение сегментов по языковым командам
- Отслеживание статуса каждого сегмента в пайплайне
- Управление очерёдностью обработки
- Сбор финальных версий и компоновка томов
- Ведение журнала прогресса проекта
- Обработка конфликтов и эскалация проблем

**Протоколы управления:**
```yaml
segment_statuses:
  - QUEUED          # В очереди на перевод
  - TRANSLATING     # Переводится
  - TRANSLATED      # Переведён (черновик)
  - EDITING         # На редактуре
  - EDITED          # Отредактирован
  - VERIFYING       # На верификации исламского контента
  - VERIFIED        # Верифицирован
  - QA_CHECK        # На финальной проверке
  - APPROVED        # Одобрен
  - AUDIO_PREP      # Подготовка к озвучке
  - COMPLETED       # Завершён
```

**Метаданные сегмента:**
```json
{
  "segment_id": "v01_ch03_s015",
  "volume": 1,
  "chapter": 3,
  "segment_number": 15,
  "arabic_word_count": 750,
  "status": {
    "ru": "EDITING",
    "uz": "TRANSLATING",
    "tj": "QUEUED"
  },
  "flags": ["HADITH_VERIFY", "COMPLEX_ISNAD"],
  "assigned_at": "2026-05-13T10:00:00Z",
  "deadline": "2026-05-14T10:00:00Z"
}
```

### 3.2 Верификатор исламского контента (ISLAMIC-VERIFIER)

**Роль:** Проверка достоверности исламского содержания во всех переводах.

**Задачи:**
- Верификация аятов Корана: сверка с оригиналом и нумерацией
- Проверка хадисов: источник, номер, степень достоверности (сахих/хасан/даиф)
- Валидация иснадов: проверка имён передатчиков и их порядка
- Проверка исторических дат (хиджра ↔ григорианский календарь)
- Верификация имён сахабов, учёных, халифов
- Проверка географических названий (древние ↔ современные)
- Контроль точности цитат из других источников (Тафсир ат-Табари и др.)

**Ресурсы для верификации:** (подробно — см. VERIFICATION_RESOURCES.md)

**Уровни верификации:**
| Уровень | Что проверяется | Критичность |
|---------|-----------------|-------------|
| L1 — Коранический | Точность аятов, нумерация сур | 🔴 Критическая — ошибки недопустимы |
| L2 — Хадисный | Текст хадиса, источник, степень | 🔴 Критическая |
| L3 — Иснадный | Имена передатчиков, последовательность | 🟡 Высокая |
| L4 — Исторический | Даты, события, генеалогии | 🟡 Высокая |
| L5 — Географический | Названия мест, локализация | 🟢 Средняя |
| L6 — Терминологический | Корректность богословских терминов | 🟡 Высокая |

**Выходные данные:**
```
- Отчёт верификации с пометками: ✅ VERIFIED / ⚠️ NEEDS_REVIEW / ❌ ERROR
- Для каждой проблемы: описание, ссылка на источник, рекомендация по исправлению
- Статистика верификации сегмента
```

### 3.3 QA агент (QA-AGENT)

**Роль:** Финальная проверка качества перед публикацией.

**Задачи:**
- Проверка форматирования: Markdown-разметка, заголовки, сноски
- Проверка полноты: все ли сегменты оригинала переведены
- Кросс-языковая проверка: совпадение нумерации глав/разделов между языками
- Проверка метаданных: номера томов, глав, страниц
- Обнаружение пропусков и дублирований
- Валидация ссылок и перекрёстных ссылок
- Проверка единообразия оформления формул благопожеланий
- Финальная сверка терминологии с глоссарием

**Чек-лист QA:**
```markdown
## Чек-лист финальной проверки сегмента

### Формат
- [ ] Корректная Markdown-разметка
- [ ] Правильная иерархия заголовков
- [ ] Номер тома/главы/сегмента указан
- [ ] Арабский текст в оригинальном написании (при наличии)

### Содержание
- [ ] Полнота перевода (нет пропущенных абзацев)
- [ ] Все аяты с указанием суры и номера
- [ ] Все хадисы с указанием источника
- [ ] Формулы благопожеланий в стандартном формате

### Терминология
- [ ] Все термины из глоссария
- [ ] Единообразие транслитерации
- [ ] Имена собственные стандартизированы

### Верификация
- [ ] Отчёт верификатора получен
- [ ] Все флаги ❌ ERROR исправлены
- [ ] Все ⚠️ NEEDS_REVIEW рассмотрены
```

### 3.4 Агент для аудио-подготовки (AUDIO-PREP-AGENT)

**Роль:** Подготовка текста к озвучке (TTS).

**Задачи:**
- Расстановка пауз и интонационной разметки для TTS
- Разметка SSML (Speech Synthesis Markup Language) для арабских вставок
- Создание фонетических подсказок для сложных имён и терминов
- Разделение текста на аудио-сегменты оптимальной длины (3–5 минут)
- Маркировка смены тона: повествование / цитата / аят / хадис
- Подготовка метаданных для аудиокниги (главы, таймкоды)

**Формат разметки:**
```xml
<segment id="v01_ch03_s015_audio_01" duration_est="4min">
  <narrator>
    Сказал Ибн Касир, <pause duration="0.3s"/> да помилует его Аллах:
  </narrator>
  <quote type="hadith" source="Сахих аль-Бухари, 1">
    <arabic phonetic="innama-l-a'malu bin-niyyat">
      إنما الأعمال بالنيات
    </arabic>
    <pause duration="0.5s"/>
    «Поистине, дела оцениваются по намерениям»
  </quote>
  <narrator>
    Этот хадис передал <name phonetic="Abu Hurayra">Абу Хурайра</name>,
    <pause duration="0.2s"/> да будет доволен им Аллах.
  </narrator>
</segment>
```

## 4. Workflow и взаимодействие

### 4.1 Основной пайплайн перевода

```
┌──────────────┐
│ КООРДИНАТОР  │ ← Разбивает текст на сегменты
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌──────────────┐
│ ТЕРМИНОЛОГ   │────▶│  ПЕРЕВОДЧИК  │ ← Получает глоссарий + сегмент
│ (по языку)   │     │  (по языку)  │
└──────────────┘     └──────┬───────┘
                            │
                            ▼
                     ┌──────────────┐
                     │   РЕДАКТОР   │ ← Стилистическая правка
                     │  (по языку)  │
                     └──────┬───────┘
                            │
                            ▼
                     ┌──────────────┐
                     │ ВЕРИФИКАТОР  │ ← Проверка исламского контента
                     │  ИСЛАМСКИЙ   │
                     └──────┬───────┘
                            │
                            ▼
                     ┌──────────────┐
                     │   QA АГЕНТ   │ ← Финальная проверка
                     └──────┬───────┘
                            │
                            ▼
                     ┌──────────────┐
                     │  АУДИО-АГЕНТ │ ← Подготовка к озвучке
                     └──────┬───────┘
                            │
                            ▼
                     ┌──────────────┐
                     │  КООРДИНАТОР │ ← Сборка финальной версии
                     └──────────────┘
```

### 4.2 Циклы обратной связи

```
ПЕРЕВОДЧИК ←──── РЕДАКТОР        (возврат при существенных ошибках)
РЕДАКТОР   ←──── ВЕРИФИКАТОР     (возврат при ошибках в исламском контенте)
ПЕРЕВОДЧИК ←──── ВЕРИФИКАТОР     (возврат при грубых ошибках перевода аятов/хадисов)
ТЕРМИНОЛОГ ←──── РЕДАКТОР        (обновление глоссария)
ТЕРМИНОЛОГ ←──── ВЕРИФИКАТОР     (корректировка терминов)
QA АГЕНТ   ────▶ КООРДИНАТОР     (эскалация нерешённых проблем)
```

### 4.3 Параллельная обработка

Три языковые команды работают **параллельно** над одним и тем же сегментом:

```
Сегмент v01_ch03_s015
    ├── [RU] ТЕРМИНОЛОГ → ПЕРЕВОДЧИК → РЕДАКТОР ──┐
    ├── [UZ] ТЕРМИНОЛОГ → ПЕРЕВОДЧИК → РЕДАКТОР ──┼──▶ ВЕРИФИКАТОР → QA
    └── [TJ] ТЕРМИНОЛОГ → ПЕРЕВОДЧИК → РЕДАКТОР ──┘
```

Верификатор проверяет **все три перевода** одновременно, что позволяет выявить расхождения между языковыми версиями.

## 5. Протоколы передачи задач

### 5.1 Формат передачи между агентами

Каждая передача задачи между агентами оформляется как структурированное сообщение:

```json
{
  "task_id": "v01_ch03_s015_ru_edit",
  "from_agent": "RU-TRANSLATOR",
  "to_agent": "RU-EDITOR",
  "segment_id": "v01_ch03_s015",
  "language": "ru",
  "priority": "normal",
  "payload": {
    "translated_text": "...",
    "arabic_original": "...",
    "glossary_version": "v2.3",
    "flags": ["TERM_CHECK"],
    "translator_notes": "Неоднозначность в иснаде — два варианта прочтения имени"
  },
  "timestamp": "2026-05-13T10:30:00Z"
}
```

### 5.2 Протокол возврата на доработку

```json
{
  "task_id": "v01_ch03_s015_ru_edit",
  "action": "RETURN",
  "from_agent": "RU-EDITOR",
  "to_agent": "RU-TRANSLATOR",
  "reason": "MAJOR_CONTENT_ERROR",
  "details": "Пропущен абзац между аятами 15 и 16 суры Аль-Бакара",
  "affected_lines": [45, 52],
  "severity": "HIGH"
}
```

### 5.3 Протокол эскалации

Когда агент не может самостоятельно разрешить проблему:

```json
{
  "action": "ESCALATE",
  "from_agent": "ISLAMIC-VERIFIER",
  "to_agent": "PROJECT-COORDINATOR",

  "options": [
    "Использовать версию Дар ас-Салям (более распространённое издание)",
    "Использовать версию Дар Ихья ат-Турас (ближе к рукописям)",
    "Указать оба варианта в сноске"
  ],
  "recommendation": "Вариант 3 — указать оба варианта",
  "references": ["shamela.ws/book/12345", "islamweb.net/hadith/67890"]
}
```

## 6. Критерии оценки работы каждого агента

### 6.1 Метрики переводчиков

| Метрика | Описание | Целевой показатель |
|---------|----------|-------------------|
| Полнота перевода | % переведённого текста от оригинала | 100% |
| Терминологическое соответствие | % терминов из глоссария | ≥ 95% |
| Коэффициент возвратов | % сегментов, возвращённых редактором | ≤ 10% |
| Скорость | Слов в час | 500–800 (арабский оригинал) |
| Точность аятов | Корректность передачи коранических текстов | 100% |
| Точность хадисов | Корректность текста и атрибуции | ≥ 99% |

### 6.2 Метрики редакторов

| Метрика | Описание | Целевой показатель |
|---------|----------|-------------------|
| Стилистическая согласованность | Оценка единообразия стиля (1–10) | ≥ 8 |
| Грамматическая корректность | % текста без грамматических ошибок | ≥ 99% |
| Коэффициент возвратов от верификатора | % сегментов, возвращённых верификатором | ≤ 5% |
| Объём правок | Среднее количество правок на 1000 слов | 15–40 (норма) |

### 6.3 Метрики терминологов

| Метрика | Описание | Целевой показатель |
|---------|----------|-------------------|
| Охват глоссария | % уникальных терминов текста в глоссарии | ≥ 98% |
| Скорость обновления | Время добавления нового термина | < 2 мин |
| Межъязыковая согласованность | Совпадение терминов RU/UZ/TJ | ≥ 90% |
