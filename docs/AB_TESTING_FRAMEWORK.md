## Зачем A/B тестирование агентов?

В проекте перевода работает **13 агентов** (3 языка × 3 роли + 4 общих). Каждый агент можно настроить по-разному:
- Разные промпты (стиль перевода, уровень формальности)
- Разные модели (Claude Sonnet vs Opus vs GPT-4o)
- Разные контексты (с глоссарием / без, с примерами / без)
- Разные температуры (0 vs 0.3 vs 0.7)

A/B тестирование позволяет **объективно выбрать лучший вариант** на основе метрик, а не субъективных впечатлений.

## 1. МЕТОДОЛОГИЯ
## ════════════════════════════════════════════════════════

### 1.1 Что тестируем

| Переменная | Варианты | Влияет на |
|-----------|----------|-----------|
| **Промпт переводчика** | v1 (базовый) vs v2 (с примерами) vs v3 (с chain-of-thought) | Точность, стиль |
| **Модель** | Claude Sonnet vs Claude Opus vs GPT-4o | Качество, стоимость, скорость |
| **Температура** | 0.0 vs 0.3 vs 0.7 | Креативность vs точность |
| **Контекст** | Только глоссарий vs Глоссарий + предыдущая глава vs Полный контекст | Консистентность |
| **Промпт редактора** | Минимальная правка vs Глубокая переработка | Стиль, сохранение авторского голоса |
| **Язык-источник** | Арабский → целевой vs Арабский → английский → целевой | Точность, нюансы |

### 1.2 Эталонный набор данных (Golden Dataset)

Создать набор из **10 эталонных фрагментов** для тестирования:

```json
{
  "version": "1.0",
  "items": [
    {
      "id": "GD-001",
      "source_arabic": "فقال ابن كثير قلت: ...",
      "source_page": "Том 1, стр. 45",
      "difficulty": "easy",
      "category": "narrative",
      "features": ["author_comment", "quran_verse", "companion_name"],
      "expected_ru": "Ибн Касир сказал: Я говорю: ...",
      "expected_terms": {
        "قلت": "Я говорю:",
        "رضي الله عنه": "(да будет доволен им Аллах)"
      }
    },
    {
      "id": "GD-002",
      "source_arabic": "...",
      "difficulty": "hard",
      "category": "theological",
      "features": ["isnad", "hadith_reference", "scholarly_debate"]
    }
  ]
}
```

**Категории фрагментов:**
1. **easy/narrative** — простое повествование
2. **medium/hadith** — хадисы с иснадами
3. **medium/quran** — аяты с тафсиром
4. **hard/theological** — богословские дискуссии
5. **hard/poetry** — стихи (касыды)

### 1.3 Процесс тестирования

```
┌─────────────────────────────────────────────────────────┐
│                   ПРОЦЕСС A/B ТЕСТА                      │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. Выбрать переменную для тестирования                 │
│     ↓                                                    │
│  2. Подготовить 2-3 варианта (A, B, C)                  │
│     ↓                                                    │
│  3. Перевести 10 эталонных фрагментов каждым вариантом  │
│     ↓                                                    │
│  4. Автоматическая оценка (скрипт + LLM-as-Judge)      │
│     ↓                                                    │
│  5. Ручная оценка (слепой обзор — без знания варианта)  │
│     ↓                                                    │
│  6. Сравнить метрики                                     │
│     ↓                                                    │
│  7. Выбрать победителя                                   │
│     ↓                                                    │
│  8. Задокументировать результат                          │
│     ↓                                                    │
│  9. Применить к production промпту                       │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 2. МЕТРИКИ СРАВНЕНИЯ АГЕНТОВ
## ════════════════════════════════════════════════════════

### 2.1 Автоматические метрики

| Метрика | Как измеряется | Вес |
|---------|---------------|-----|
| **Терминологическая точность** | % совпадения с глоссарием | 25% |
| **Полнота** | % переведённых предложений | 20% |
| **Языковая чистота** | Отсутствие дрифта (L001-L003) | 15% |
| **Формат** | Правильные даты, транслитерация | 10% |
| **Скорость** | Токены/сек, общее время | 5% |
| **Стоимость** | $ за 1000 слов перевода | 5% |

### 2.2 LLM-as-Judge метрики (адаптация из hadith-verifier)

| Метрика | Описание | Вес |
|---------|----------|-----|
| **Accuracy (Точность)** | Смысл оригинала передан верно | 25% |
| **Style (Стиль)** | Сохранён авторский голос, естественный язык | 10% |
| **Terminology (Терминология)** | Богословские термины корректны | 10% |

### 2.3 Ручные метрики (слепая оценка)

| Метрика | Оценщик | Шкала |
|---------|---------|-------|
| **Читаемость** | Носитель языка | 1-5 |
| **Богословская точность** | Исламовед | 1-5 |
| **Естественность** | Носитель языка | 1-5 |
| **Предпочтение** | A vs B vs C (принудительный выбор) | Ранг |

### 2.4 Итоговая формула

```
Score = (auto_metrics × 0.50) + (llm_judge × 0.25) + (human_eval × 0.25)
```

Побеждает вариант с наивысшим Score. При разнице < 5% — выбирается более дешёвый.

## 3. ТЕСТИРОВАНИЕ ПОДХОДОВ К ПЕРЕВОДУ
## ════════════════════════════════════════════════════════

### Тест 1: Прямой vs двухэтапный перевод

| Вариант A | Вариант B |
|-----------|-----------|
| Арабский → Русский (напрямую) | Арабский → Английский → Русский |
| Плюс: сохраняет нюансы арабского | Плюс: LLM лучше работает с английским |
| Минус: может быть неестественно | Минус: двойная потеря нюансов |

**Гипотеза:** Для простого нарратива A лучше, для богословских текстов B лучше.

### Тест 2: Chain-of-Thought vs прямой промпт

| Вариант A (прямой) | Вариант B (CoT) |
|--------------------|-----------------|
| `Переведи текст на русский` | `1. Определи ключевые термины. 2. Найди аяты/хадисы. 3. Переведи, сохраняя стиль.` |
| Быстрее, дешевле | Точнее для сложных фрагментов |

### Тест 3: Температура

| T=0.0 | T=0.3 | T=0.7 |
|-------|-------|-------|
| Детерминированный | Слегка вариативный | Креативный |
| Для терминологии | Для нарратива | Для поэзии |

### Тест 4: Размер контекста

| Минимум | Средний | Максимум |
|---------|---------|----------|
| Только глоссарий (2K токенов) | + предыдущая страница (8K) | + вся глава + смежные главы (32K+) |
| Быстро, дёшево | Баланс | Консистентно, но дорого |

## 4. ПРОЦЕСС ВЫБОРА ЛУЧШЕГО ВАРИАНТА
## ════════════════════════════════════════════════════════

### Шаг 1: Определить гипотезу
```
Гипотеза: Промпт v2 (с примерами) даёт более высокую терминологическую 
точность, чем промпт v1 (базовый) при переводе богословских текстов.
```

### Шаг 2: Провести тест
- Минимум **10 фрагментов** из Golden Dataset
- Каждый фрагмент переводится **3 раза** каждым вариантом (для оценки стабильности)
- Итого: 10 × 3 × 2 = 60 переводов на один A/B тест

### Шаг 3: Собрать метрики
```json
{
  "test_id": "ABT-001",
  "date": "2026-05-15",
  "variable": "prompt_version",
  "variant_a": "v1_basic",
  "variant_b": "v2_with_examples",
  "fragments_tested": 10,
  "runs_per_fragment": 3,
  "results": {
    "variant_a": {
      "terminology_accuracy": 0.89,
      "completeness": 1.0,
      "language_purity": 0.95,
      "llm_judge_avg": 0.78,
      "human_preference_wins": 3,
      "cost_per_1k_words": 0.045,
      "avg_time_seconds": 12.3
    },
    "variant_b": {
      "terminology_accuracy": 0.96,
      "completeness": 1.0,
      "language_purity": 0.97,
      "llm_judge_avg": 0.88,
      "human_preference_wins": 7,
      "cost_per_1k_words": 0.052,
      "avg_time_seconds": 15.1
    }
  },
  "winner": "variant_b",
  "confidence": "high",
  "decision": "Принять v2. Рост точности +7% оправдывает рост стоимости +15%"
}
```

### Шаг 4: Принять решение

| Разница в Score | Решение |
|-----------------|---------|
| > 10% | Явный победитель, применить немедленно |
| 5-10% | Победитель, но проверить на 20 дополнительных фрагментах |
| < 5% | Статистически незначимо. Выбрать более дешёвый/быстрый |

### Шаг 5: Задокументировать

Добавить запись в **AB_TEST_LOG** (см. раздел 5).

## 7. ПРОМПТ ДЛЯ LLM-СУДЬИ
## ════════════════════════════════════════════════════════

```
Ты — эксперт по оценке качества перевода исламских исторических текстов.
Тебе даны ОРИГИНАЛ (арабский) и ДВА перевода (A и B).
Ты НЕ знаешь, какой вариант какой. Оцени ОБЪЕКТИВНО.

Критерии (каждый 0.0–1.0):
1. accuracy — смысл оригинала передан верно
2. completeness — ничего не пропущено, ничего не добавлено
3. terminology — богословские термины корректны
4. style — естественный язык, сохранён авторский голос
5. formatting — даты, имена, ссылки оформлены правильно

Оригинал: {original}
Глоссарий: {glossary}
Перевод A: {translation_a}
Перевод B: {translation_b}

Верни ТОЛЬКО JSON:
{
  "a": {"accuracy":0,"completeness":0,"terminology":0,"style":0,"formatting":0,"overall":0},
  "b": {"accuracy":0,"completeness":0,"terminology":0,"style":0,"formatting":0,"overall":0},
  "winner": "a|b|tie",
  "reasoning": "одно предложение"
}
```

**Правила судьи (из hadith-verifier):**
- temperature=0 для детерминированности
- Использовать ДРУГУЮ модель, чем переводчик (избежать bias)
- Запускать 3 раза, брать медиану
