1. **Перед началом перевода** — агент сканирует этот файл на релевантные паттерны

4. **Еженедельно** — Координатор анализирует частоту паттернов и обновляет промпты агентов

## ПАТТЕРН T001: Неконсистентная транслитерация имён
**ID:** T001  
**Тип:** Терминология  
**Симптом:**
  - Одно и то же имя транслитерировано по-разному в разных главах
  - Пример: «Абу Хурайра» vs «Абу Хурейра» vs «Абу Хурайрах»

**Причина:**
  Агент-переводчик не сверился с глоссарием перед работой.
  LLM обучен на разных источниках с разной транслитерацией.

**Исправление:**
  1. Перед началом перевода загрузить актуальный глоссарий в контекст
  2. Добавить в промпт: `ОБЯЗАТЕЛЬНО: Используй следующую транслитерацию имён: {glossary}`
  3. Автопроверка: скрипт сравнивает имена с глоссарием после каждой главы

**Урок:** Негативные + позитивные примеры в промпте работают лучше общих инструкций (P003 из hadith-verifier)

**Статус:** АКТИВНЫЙ — применять всегда

---

## ПАТТЕРН T002: Смешение переводов богословских терминов
**ID:** T002  
**Тип:** Терминология  
**Симптом:**
  - «таухид» → в одном месте «единобожие», в другом «монотеизм»
  - «ширк» → «многобожие» vs «идолопоклонство» vs «придание сотоварищей»

**Причина:**
  Каждый термин имеет несколько допустимых переводов.
  LLM выбирает случайно без фиксации.

**Исправление:**
  1. Фиксировать ОДИН перевод для каждого термина в глоссарии
  2. В промпт: `Для термина "таухид" всегда используй "единобожие". НЕ используй "монотеизм".`
  3. Допустимые альтернативы указывать в сносках, но не в основном тексте

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН T003: Калькирование арабских конструкций
**ID:** T003  
**Тип:** Стилистика  
**Симптом:**
  - «И было дело таким, что…» (арабская конструкция فكان الأمر كذلك)
  - «Воистину, поистине, в действительности» — нагромождение усилительных частиц

**Причина:**
  LLM при переводе с арабского часто сохраняет синтаксические конструкции оригинала.

**Исправление:**
  1. Добавить в промпт редактора: `Перестрой предложения в естественный русский синтаксис. Избегай калькирования арабского порядка слов.`
  2. Автопроверка: найти повторяющиеся «Воистину», «Поистине» > 3 раз на страницу

**Статус:** АКТИВНЫЙ

## ПАТТЕРН L001: Таджикские слова в узбекском тексте
**ID:** L001  
**Тип:** Языковой дрифт  
**Аналог в hadith-verifier:** P003 (Uzbek in Tajik output)  
**Симптом:**
  - Узбекский перевод содержит «ин китоб» (тадж.) вместо «бу китоб» (узб.)
  - Последние предложения главы «дрифтуют» в таджикский

**Причина:**
  LLM обучен на смешанных узбекских/таджикских корпусах.
  Обе языка используют кириллицу и имеют общую исламскую лексику.

**Исправление:**
```
ОБЯЗАТЕЛЬНО: Пиши ТОЛЬКО на узбекском кириллическом языке.
НЕ используй таджикские слова: "ин" (используй "бу"), "аст" (используй "дир"),
"барои" (используй "учун"), "китоб" (используй "китоб" с узб. грамматикой).
КАЖДОЕ предложение — включая последнее — должно быть на узбекском.
```

**Автопроверка:**
```python
TAJIK_MARKERS = ['ӯ', 'ҷ', 'ӣ']  # Символы, отсутствующие в узбекском
for char in text:
    if char in TAJIK_MARKERS:
        flag_error(f"Таджикский символ '{char}' обнаружен в узбекском тексте")
```

**Статус:** АКТИВНЫЙ

## ПАТТЕРН L002: Узбекские слова в таджикском тексте
**ID:** L002  
**Тип:** Языковой дрифт  
**Аналог в hadith-verifier:** P003  
**Симптом:**
  - Таджикский текст содержит «қилинг» вместо «кунед»
  - Запрещённые фразы: «ташриф буюринг», «марҳамат қилинг»

**Исправление:**
```
НЕ используй узбекские слова: "ташриф буюринг", "марҳамат қилинг".
Используй таджикские: "барои дидан гузаред", "ба манба муроҷиат кунед".
При упоминании Пророка пиши (с.а.в.), НЕ (с).
```

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН L003: Латиница в кириллическом тексте
**ID:** L003  
**Тип:** Скрипт  
**Аналог в hadith-verifier:** P018  
**Симптом:**
  - Буква «c» (латинская) вместо «с» (кириллическая) — визуально идентичны
  - Буква «o» (латинская) вместо «о» (кириллическая)

**Причина:**
  Copy-paste из разных источников смешивает кодовые страницы.

**Исправление:**
```python
CONFUSABLES = {
    'a': 'а', 'e': 'е', 'o': 'о', 'p': 'р', 'c': 'с',
    'x': 'х', 'y': 'у', 'A': 'А', 'B': 'В', 'C': 'С',
    'E': 'Е', 'H': 'Н', 'K': 'К', 'M': 'М', 'O': 'О',
    'P': 'Р', 'T': 'Т', 'X': 'Х',
}
for latin, cyrillic in CONFUSABLES.items():
    text = text.replace(latin, cyrillic)
```

**Статус:** АВТОМАТИЗИРОВАН

## ПАТТЕРН I001: Неверный номер аята
**ID:** I001  
**Тип:** Верификация — CRITICAL  
**Симптом:**
  - Аят приведён с неправильным номером суры или аята
  - Перевод аята не совпадает с эталонным

**Причина:**
  LLM «галлюцинирует» номера аятов (аналог P010 из hadith-verifier — hallucinated hadith URLs).

**Исправление:**
  1. Всегда сверять номера аятов с quran.com API или локальной БД
  2. В промпт: `Не придумывай номера аятов. Если не уверен — оставь [ПРОВЕРИТЬ: сура X, аят Y]`
  3. Автопроверка: regex для формата «(Сура Название, X:Y)» + сверка с БД

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН I002: Отсутствие салавата при упоминании Пророка ﷺ
**ID:** I002  
**Тип:** Верификация — HIGH  
**Симптом:**
  - «Пророк сказал…» без «ﷺ» или «(мир ему и благословение Аллаха)»

**Исправление:**
  1. Автопроверка: regex на «Пророк[^ﷺ]» без следующего салавата в пределах 5 символов
  2. В промпт: `КАЖДОЕ упоминание Пророка Мухаммада ОБЯЗАТЕЛЬНО сопровождать ﷺ или (мир ему и благословение Аллаха)`

**Статус:** АВТОМАТИЗИРОВАН

---

## ПАТТЕРН I003: Пропуск «разия Ллаху анху» для сподвижников
**ID:** I003  
**Тип:** Верификация — MEDIUM  
**Симптом:**
  - «Абу Бакр сказал…» без «(да будет доволен им Аллах)»

**Исправление:**
  1. Список известных сподвижников в глоссарии с пометкой «требуется дуа»
  2. Автопроверка: поиск имён сподвижников без следующего «(р.а.)» или развёрнутой формулы

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН I004: Смешение хадисов из разных сборников
**ID:** I004  
**Тип:** Верификация — CRITICAL  
**Симптом:**
  - Хадис приписан Аль-Бухари, но на самом деле из Муслима
  - Номер хадиса не соответствует указанному сборнику

**Причина:**
  LLM генерирует правдоподобные, но несуществующие номера (P010 hadith-verifier).

**Исправление:**
  1. Использовать sunnah.com API для верификации: `GET https://api.sunnah.com/v1/hadiths/{collection}:{number}`
  2. Если API не подтверждает — помечать [ВЕРИФИЦИРОВАТЬ ВРУЧНУЮ]
  3. В промпт: `Используй общие ссылки на сборники, не придумывай конкретные номера хадисов`

**Статус:** АКТИВНЫЙ

## ПАТТЕРН D001: UTF-8 BOM в файлах
**ID:** D001  
**Тип:** Кодировка  
**Аналог в hadith-verifier:** P020  
**Симптом:**
  - Файл открывается с артефактами в начале
  - Git diff показывает изменение в первой строке без видимых различий

**Исправление:**
  - Сохранять все файлы в UTF-8 без BOM
  - Git hook: `file --mime-encoding` проверка при коммите

**Статус:** АВТОМАТИЗИРОВАН

---

## ПАТТЕРН D002: Пропущенные абзацы при длинных главах
**ID:** D002  
**Тип:** Полнота — HIGH  
**Симптом:**
  - Подсчёт абзацев оригинала ≠ подсчёт абзацев перевода
  - Обычно пропускаются абзацы в середине длинных глав

**Причина:**
  Контекстное окно LLM переполняется, и модель «теряет» части текста.

**Исправление:**
  1. Разбивать главы на сегменты по 1000-1500 слов
  2. Каждый сегмент переводить отдельно с перекрытием в 2-3 предложения
  3. Автопроверка: hash-сравнение количества абзацев оригинал vs перевод

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН D003: Дубликаты абзацев
**ID:** D003  
**Тип:** Полнота — MEDIUM  
**Симптом:**
  - Один и тот же абзац переведён дважды
  - Часто на стыке сегментов (из-за перекрытия в D002)

**Исправление:**
  1. Fuzzy-matching на стыках сегментов (similarity > 0.85 = дубликат)
  2. Автопроверка при склейке сегментов

**Статус:** АКТИВНЫЙ

---

## ПАТТЕРН D004: Неконсистентное форматирование дат
**ID:** D004  
**Тип:** Форматирование  
**Симптом:**
  - «132 г.х.» vs «132 год хиджры» vs «132 х. / 750 г.» vs «132/750»

**Исправление:**
  Стандартный формат: **132 г.х. / 750 г.** (или развёрнуто при первом упоминании в главе)
  Regex: `\d+ г\.х\. / \d+ г\.`

**Статус:** АКТИВНЫЙ

## ПАТТЕРН S001: Монотонный стиль (все предложения одной длины)
**ID:** S001  
**Тип:** Стилистика  
**Симптом:**

  В промпт редактора: `Варьируй длину предложений: чередуй короткие (5-10 слов) и длинные (20-30 слов). Используй риторические вопросы для вовлечения.`

## ПАТТЕРН S002: Потеря авторского голоса Ибн Касира
**ID:** S002  
**Тип:** Стилистика — HIGH  
**Симптом:**
  - Перевод звучит как современный учебник, а не как классическое произведение
  - Потеряны авторские комментарии типа «قلت» (Я говорю / Замечу)

**Исправление:**
  1. Сохранять авторские вставки: «Я говорю:», «Замечу:», «Ибн Касир комментирует:»
  2. В промпт: `Сохрани авторский стиль Ибн Касира — его личные комментарии, вводные слова, структуру рассуждений`

**Статус:** АКТИВНЫЙ

### Ключевые уроки из hadith-verifier (35 паттернов, 83+ CI прогонов)

| Урок | Паттерн-источник | Применение к переводу |
|------|-----------------|----------------------|
| Негативные + позитивные примеры > общие инструкции | P003 | Давать агентам конкретные «НЕ используй X, используй Y» |
| Всегда тестировать ВСЕ пути кода | P035, P041 | Проверять все 3 языка при каждом изменении промпта |
| LLM галлюцинирует номера и ссылки | P010 | Все номера аятов/хадисов верифицировать через API/БД |
| Не утверждать точные значения — утверждать диапазоны | P014 | Не ожидать конкретную формулировку — проверять семантику |
| CI ловит только то, что тестируется | P035 | Каждый языковой путь должен иметь проверку |
| Один провальный тест не должен блокировать весь pipeline | P011 | Разделять блокирующие (CRITICAL) и предупреждающие проверки |
| UTF-8 BOM — тихий убийца | P020 | Всегда сохранять без BOM, проверять при коммите |
| Языковой дрифт усиливается к концу текста | P003 | Особое внимание последним абзацам каждой главы |
