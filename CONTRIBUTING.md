## 🕌 Кодекс поведения

Этот проект является **садакой джарией** (непрерывным благодеянием). Мы ожидаем от всех участников:

1. **Уважение** — к другим участникам, к оригинальному тексту и к читателям
2. **Точность** — приоритет смысловой точности перед литературной красотой
3. **Скромность** — признание ошибок и открытость к исправлениям
4. **Ответственность** — понимание важности работы с религиозным текстом

## 🔄 Процесс работы

### Workflow

```
1. Выбрать Issue → 2. Создать ветку → 3. Перевести/Редактировать
→ 4. Проверить качество → 5. Создать PR → 6. Code Review → 7. Merge
```

### Шаг за шагом

1. **Создайте ветку** от `main`:
   ```bash
   git checkout -b translate/vol01-ru-chapter-001
   ```

2. **Работайте** над переводом, коммитя поглавно

3. **Проверьте** перед пушем:
   - Кодировка файлов — UTF-8
   - Формат — Markdown
   - Терминология — согласно глоссарию

4. **Запушьте** и создайте Pull Request

## 📝 Правила перевода

### Общие принципы

1. **Смысловой перевод** — передача смысла, а не буквальный перевод
2. **Единая терминология** — использование глоссария из `glossary/`
3. **Аяты Корана** — использование только признанных переводов смыслов
4. **Хадисы** — сохранение иснада (цепочки передатчиков)
5. **Транслитерация** — по стандартам, определённым в Конституции проекта

### Формат файлов

- **Кодировка**: UTF-8 (без BOM)
- **Формат**: Markdown (.md)
- **Именование**: `chapter_NNN.md` (например, `chapter_001.md`)
- **Структура главы**:

```markdown
# Глава N: Название главы

## Оригинальный текст (арабский)
...

## Перевод
...

## Примечания переводчика
...
```

### Языковые особенности

| Аспект | Русский | Узбекский | Таджикский |
|--------|---------|-----------|------------|
| Алфавит | Кириллица | Кириллица | Кириллица |
| Стиль | Научно-популярный | Научно-популярный | Научно-популярный |
| Обращение | «вы» к читателю | Сиз | Шумо |

## 🌿 Именование веток

```
Формат: <тип>/<том>-<язык>-<описание>

Примеры:
translate/vol01-ru-chapters-1-5     # Перевод глав 1-5 тома 1 на русский
edit/vol01-uz-chapter-003           # Редактура главы 3 тома 1 на узбекский
glossary/update-islamic-terms       # Обновление глоссария
fix/vol01-ru-hadith-references      # Исправление ссылок на хадисы
docs/update-qa-standards            # Обновление документации QA
```

## 💬 Коммиты

Используем формат [Conventional Commits](https://www.conventionalcommits.org/):

```
<тип>(<область>): <описание>

Типы:
translate  — новый перевод
edit       — редактура существующего перевода
verify     — верификация исламского контента
fix        — исправление ошибок
glossary   — обновление глоссария
docs       — обновление документации
ci         — изменения CI/CD
audio      — аудио-связанные изменения

Примеры:
translate(vol01-ru): chapters 1-3 draft translation
edit(vol01-uz): improve terminology in chapter 5
verify(vol01-ru): validate hadith references in chapter 2
glossary(ru): add 50 new Islamic terms
```

## 📬 Pull Requests

### Шаблон PR

При создании PR используйте следующий шаблон:

```markdown
## Описание
[Краткое описание изменений]

## Тип изменений
- [ ] Новый перевод
- [ ] Редактура
- [ ] Верификация
- [ ] Исправление ошибки
- [ ] Обновление глоссария
- [ ] Документация

## Чеклист
- [ ] Прочитал CONTRIBUTING.md
- [ ] Файлы в кодировке UTF-8
- [ ] Терминология соответствует глоссарию
- [ ] Аяты Корана проверены
- [ ] Хадисы с указанием источника
- [ ] Нет грамматических ошибок
```

### Правила PR

1. **Один PR = одна логическая единица** (одна глава, один набор исправлений)
2. **Описательный заголовок** на английском или русском
3. **Ссылка на Issue**, если применимо
4. **Не более 5 глав** в одном PR

## 🔍 Обзор кода (Code Review)

### Кто проверяет

| Тип изменений | Обязательные ревьюеры |
|---------------|----------------------|
| Перевод | Редактор + Терминолог |
| Терминология | Терминолог + Верификатор |
| Исламский контент | Верификатор |
| Скрипты/CI | Мейнтейнер |

### Критерии одобрения

- ✅ Минимум 1 approval от ревьюера
- ✅ Все автоматические проверки пройдены
- ✅ Нет конфликтов с `main`
- ✅ Терминология согласована
